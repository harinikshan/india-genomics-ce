# Security Policy

## Supported release
- `v1.0.0-oss` (Community Edition)

## Reporting a vulnerability
Please report security vulnerabilities privately before public disclosure.

Until a dedicated security contact is published, open a private report via your
project's secure contact channel and include:
- Affected version/commit
- Reproduction steps
- Impact assessment
- Suggested remediation (optional)

## Scope
Security issues include, but are not limited to:
- Path traversal
- Arbitrary file read/write
- Injection and XSS
- Sensitive data leakage
- Authentication/authorization flaws (if enabled)
